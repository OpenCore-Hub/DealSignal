## 1. 全局交互原则

### 1.1 即时反馈

每个用户操作必须在 100ms 内得到视觉反馈：
- 按钮按下：scale 0.98
- 悬停：背景色变化
- 加载：Skeleton 或 spinner
- 完成：toast 通知

### 1.2 渐进披露

复杂功能分步骤展示：
- 默认展示核心控制
- 高级选项折叠
- 上下文帮助按需出现

### 1.3 减少认知负荷

- 每个页面最多 3 个主要操作
- 使用标签页组织复杂内容
- 信号卡片不超过 4 行信息

### 1.4 一致性

- 相同交互使用相同组件
- 相同状态使用相同颜色
- 相同操作使用相同文案

## 2. 交易雷达 Dashboard

### 2.1 信号流 Signal Stream

#### 2.1.1 卡片结构

```
┌─────────────────────────────────────────────────────────┐
│ [图标]  [对象名称]                              [热度标] │
│         行为描述（一行）                                │
│         💡 系统解释                                     │
│         [建议动作]  [次要动作]  [忽略]  [稍后]           │
└─────────────────────────────────────────────────────────┘
```

#### 2.1.2 交互行为

**悬停**：
- 卡片背景从 `--card` 变为 `--muted`
- 显示完整建议动作按钮

**点击**：
- 展开卡片，显示完整时间线
- 再次点击折叠

**滑动（移动端）**：
- 左滑：归档/忽略
- 右滑：标记已跟进
- 滑动距离超过 40% 触发动作

**动作按钮**：
- 主按钮：直接执行最高优先级建议
- 次按钮：查看联系人/文档详情
- 忽略：从当前视图移除，进入已归档
- 稍后：1 小时后再次提醒

#### 2.1.3 进入动效

```css
/* 信号流卡片 stagger 进入 */
@keyframes signalEnter {
  from { opacity: 0; transform: translateY(12px); }
  to { opacity: 1; transform: translateY(0); }
}
/* 每个卡片延迟 50ms */
```

### 2.2 热度地图 Heat Map

#### 2.2.1 网格布局

- 桌面：3 列
- 平板：2 列
- 移动：1 列

#### 2.2.2 卡片状态

```
默认：白色背景，1px border
悬停：阴影 + 边框加粗
选中：ring-2 ring-primary
Hot：左边框 4px #ef4444
Warm：左边框 4px #f59e0b
Cold：左边框 4px #3b82f6
Risk：顶部边框 4px #a855f7
```

#### 2.2.3 悬停浮层

悬停 300ms 后显示 tooltip：
- 文档标题
- 总访问次数
- 平均停留时长
- 热度分
- Top 1 关注页面

### 2.3 行动队列 Action Queue

#### 2.3.1 任务卡片

```
┌─────────────────────────────────────────┐
│ [优先级] 任务描述                        │
│          预计影响：高 | 截止：今天 18:00 │
│ [执行] [推迟] [忽略]                     │
└─────────────────────────────────────────┘
```

#### 2.3.2 优先级视觉

- High：红色左边框
- Medium：琥珀色左边框
- Low：灰色左边框

#### 2.3.3 执行流程

点击"执行"：
1. 如果是发邮件：打开邮件草稿弹窗
2. 如果是补充材料：跳转到文档/数据室
3. 如果是联系：跳转到联系人详情
4. 完成后自动归档

## 3. 智能链接创建器 Smart Link Composer

### 3.1 滑块交互

#### 3.1.1 视觉设计

```
低摩擦公开                    高安全 NDA
←―――――――――――――――――――――――――――――――――――→
●―――――――――――――――――――――――――――――――――――●
```

- 轨道背景：渐变从左（绿/蓝）到右（红/紫）
- 滑块 thumb：圆形，带当前等级数字
- 刻度标记：5 个等级

#### 3.1.2 拖动反馈

- 拖动时实时更新下方步骤列表
- 释放时轻微吸附到最近等级
- 伴有 50ms haptic feedback（如设备支持）

#### 3.1.3 等级切换动画

切换等级时：
- 步骤列表项依次淡入/淡出（stagger 30ms）
- 安全强度标签更新
- 推荐文案更新

### 3.2 接收方步骤预览

```
接收方需要完成：
✓ 点击链接
☐ 输入邮箱验证
☐ 输入密码
☐ 签署 NDA
☐ 等待白名单审批
```

- 已完成步骤：绿色勾选
- 待完成步骤：灰色圆圈
- 当前选中等级的步骤：高亮

### 3.3 推荐配置

根据文档类型自动推荐：
- PDF pitch deck → Level 2（Email）
- Financial model → Level 4（Whitelist）
- Data room → Level 5（NDA）
- Sales proposal → Level 3（Password）

推荐标签可点击一键应用。

### 3.4 链接生成

点击"创建链接"：
1. 按钮变为 loading
2. 显示生成中的 Skeleton
3. 成功后展开结果区域：
   - 短链接（可复制）
   - QR 码
   - 邮件模板选择
   - 预览链接

## 4. AI 悬浮助手 2.0

### 4.1 悬浮按钮

#### 4.1.1 位置

- 桌面：右下角，距边缘 24px
- 移动：右下角，距边缘 16px

#### 4.1.2 状态

```
默认：小圆点，AI 图标
有新摘要时： gentle pulse 动画
展开时：扩展为对话面板
```

### 4.2 对话面板

#### 4.2.1 布局

```
┌─────────────────────────────┐
│  标题 + 关闭                 │
├─────────────────────────────┤
│  自动摘要卡片                │
├─────────────────────────────┤
│  历史消息滚动区              │
│  User: ...                  │
│  AI: ... [evidence]         │
├─────────────────────────────┤
│  输入框 + 发送               │
└─────────────────────────────┘
```

#### 4.2.2 消息气泡

- 用户：右对齐，primary 背景
- AI：左对齐，muted 背景
- 最大宽度 80%

#### 4.2.3 Evidence 展示

```
💬 "根据第 12 页财务数据..."

📄 Evidence 1 — 第 12 页 · 置信度 92%
📄 Evidence 2 — 第 8 页 · 置信度 78%
```

- Evidence 可点击
- 点击后 Canvas 跳转并高亮
- 当前选中的 evidence 有边框高亮

### 4.3 Canvas 高亮框

#### 4.3.1 绘制规则

- 使用 bbox 坐标换算为 Canvas 像素坐标
- 高亮框颜色：#3b82f6（默认）/ #f59e0b（次要）/ #ef4444（关键）
- 边框宽度：2px
- 圆角：4px

#### 4.3.2 动画

```css
@keyframes highlightPulse {
  0%, 100% { box-shadow: 0 0 0 0 rgba(59, 130, 246, 0.4); }
  50% { box-shadow: 0 0 0 8px rgba(59, 130, 246, 0); }
}
/* 持续 2s，循环 3 次后停止 */
```

#### 4.3.3 跳转

- 目标页未加载：先加载页面，再绘制高亮
- 目标页已加载：直接平移/缩放到高亮区域
- 过渡动画：300ms ease-out

### 4.4 向发件人提问

接收方点击"向发件人提问"：
1. 输入问题
2. 发送后显示"已通知发件人"
3. 发件人 Dashboard 收到 Signal
4. 发件人回复后接收方在 Viewer 内收到通知

## 8. 动效规范

### 8.1 时长

| 场景 | 时长 |
|---|---|
| 按钮反馈 | 100ms |
| 颜色过渡 | 150ms |
| 页面切换 | 200ms |
| 卡片进入 | 300ms |
| Canvas 跳转 | 300ms |
| 高亮 pulse | 2000ms |

### 8.2 缓动

- 默认：`cubic-bezier(0.16, 1, 0.3, 1)`
- 弹性（仅用于特殊场景）：`cubic-bezier(0.34, 1.56, 0.64, 1)`

### 8.3 减少动效

- 尊重 `prefers-reduced-motion`
- 关闭非必要动画
- 保留必要的状态变化

## 9. 无障碍规范

### 9.1 键盘导航

- Tab 顺序符合视觉顺序
- 所有交互元素可聚焦
- 焦点状态可见（ring-2 ring-ring）

### 9.2 屏幕阅读器

- 信号卡片有完整 aria-label
- 图表有替代文本
- 动态内容更新通过 aria-live 通知

### 9.3 颜色

- 热/温/冷同时有文字标签
- 关键信息不依赖颜色 alone
- 对比度 ≥ 4.5:1
