# DealSignal v2.1.1 产品设计方案（精炼版）

> 基于细胞级审计的二次迭代：减少噪音、放大信号、建立信任、导向行动。

---

## 1. 设计目标

1. **Signal-First**：首屏必须回答“今天谁最热、我要做什么、有没有风险”。
2. **认知减负**：每屏主要决策组不超过 3 个，次要信息折叠或后置。
3. **数据诚实**：禁止伪造数据与占位图表，无数据时给明确空态。
4. **行动导向**：每个信号与洞察必须带一个主行动。
5. **渐进安全**：安全设置与接收方摩擦同时可视化。

## 2. 信息架构

### 2.1 全局导航（Sidebar）

| 路由 | 中文标签 | 说明 |
|------|----------|------|
| `/dashboard` | 交易雷达 | 信号流 + 待办 + 风险 |
| `/documents` | 文档库 | 文档列表与管理 |
| `/links` | 链接 | 可追踪链接列表 |
| `/deal-rooms` | 数据室 | 数据室列表与信号 |
| `/contacts` | 联系人 | 访客与意向评分 |
| `/insights` | 洞察 | 综合分析与建议 |
| `/settings` | 设置 | 工作区与账户设置 |

### 2.2 Dashboard 信息架构

```
Dashboard
├── 摘要条（高热度信号数 / 待办行动数）
├── 风险提醒条（异常 / 过期 / 高敏下载）
├── 主栏（lg:col-span-2）
│   └── 信号流（按 hot > risk > warm > cold 排序）
└── 侧栏（lg:col-span-1）
    ├── 待办行动（排序 + 上下文 + 状态治理）
    ├── 热度地图（hot/warm 可点击列表）
    └── 最近文档（折叠为 3 条）
```

### 2.3 文档详情信息架构

```
Document Detail
├── 页头（文档名 + 元信息 + 主操作：创建链接）
├── 侧边栏
│   ├── 总访问
│   ├── 独立访客
│   ├── 平均停留
│   └── 热度分布（真实数据或隐藏）
├── Tabs
│   ├── 总览：关键信号 + 最近动作 + 待办
│   ├── 内容：页面缩略图 + heat bar + 证据高亮
│   ├── 数据：页面级分析图表
│   └── AI 洞察：证据驱动的洞察 + 快捷问题
└── 链接 Tab / 折叠面板：此文档的链接
```

### 2.4 智能链接创建器信息架构

```
Smart Link Creator
├── 文档选择
├── 权限预设模板（低摩擦 / 中强度 / 高强度）
├── 自定义安全选项（默认折叠）
└── 预览面板
    ├── 安全强度条
    ├── 接收方摩擦条
    ├── 场景建议
    └── 已生成链接 + 复制反馈
```

### 2.5 Deal Rooms 信息架构

```
Deal Rooms
├── 列表页：模板标签 + 最近信号 + 待审批数
├── 新建向导
│   ├── Step 1: 选择模板
│   └── Step 2: 填写信息 + 预览结构/文件/权限
└── 详情页
    ├── 首屏：最近访问者 + 热度 + 待审批 + 行动建议
    └── 折叠区：文件夹 tree + 推荐文件清单
```

### 2.6 Contacts 信息架构

```
Contacts
├── 列表页：头像 + 姓名/邮箱 + 热度分 + 最后访问
├── 筛选：只看 hot
└── 详情页
    ├── Overview：关键指标 + 热度 + 最近活动 + 浏览文档 + 备注 + AI 建议
    ├── 活动时间线
    ├── 浏览文档
    └── 备注
```

## 3. 页面规范

### 3.1 Dashboard

**首屏目标**：5 秒内识别今日最关键信号与行动。

| 元素 | 规范 |
|------|------|
| 摘要条 | 2 个指标：高热度信号数、待办行动数。数字使用 `.text-stat`。 |
| 风险提醒 | 顶部通条，紫色 risk，左侧 Warning 图标，右侧“查看详情”。 |
| 信号流 | 主栏，最多展示 10 条，滚动加载。每条卡片左侧 4px 色条。 |
| 待办行动 | 侧栏，最多 5 条 pending，已完成折叠。 |
| 热度地图 | 侧栏，只展示 hot/warm，cold 折叠。每项展示访问数、平均停留、最后访问。 |
| 最近文档 | 页面底部，最多 3 条。 |

### 3.2 文档库

| 元素 | 规范 |
|------|------|
| 列表列 | 标题（类型+名称）、热度、最近访问、操作。 |
| 行操作 | 复制链接、创建链接、查看详情。 |
| 空态 | 无文档时引导上传。 |

### 3.3 链接列表

| 元素 | 规范 |
|------|------|
| 列表列 | 链接、文档、热度、最近访问、操作。 |
| 启用状态 | 用 Badge（已启用/已停用），不使用 Switch + Yes/No。 |
| 行操作 | 复制、查看详情。 |

### 3.4 文档详情

| 元素 | 规范 |
|------|------|
| 页头主操作 | 创建链接。预览/下载未实现时置灰 + tooltip。删除加二次确认。 |
| Tabs | 总览 / 内容 / 数据 / AI 洞察。 |
| 内容页 | 页面缩略图网格，A4 比例，底部 heat bar，选中后展示详情与证据。 |
| 数据页 | 柱状图带 tooltip、坐标标签、空态。 |
| AI 洞察 | 洞察卡片按优先级排序，展示证据片段、样本量、置信度。 |

### 3.5 智能链接创建器

| 元素 | 规范 |
|------|------|
| 权限预设 | 3 个卡片，选中态左侧 accent bar + 背景填充。 |
| 自定义区 | 默认折叠，点击“自定义安全选项”展开。 |
| 预览 | 安全/摩擦分数条 + 场景建议文案。 |
| 生成后 | 链接可复制，复制后按钮图标变 Check。 |

### 3.6 Deal Rooms

| 元素 | 规范 |
|------|------|
| 列表卡片 | 模板标签、名称、最近信号、待审批数、最后访问。 |
| 新建向导 | 两步，Step 1 只展示模板卡片，Step 2 左右分栏（表单 + 预览）。 |
| 详情首屏 | 最近访问者、热度、待审批、行动建议。 |
| 文件清单 | 真实上传状态，未上传项显示“上传”按钮，不上传时显示空态。 |

### 3.7 Contacts

| 元素 | 规范 |
|------|------|
| 列表卡片 | 头像/首字母、姓名、邮箱、热度分、最后访问。 |
| 筛选 | “只看 hot” quick filter。 |
| Overview | 关键指标、热度徽章、最近 3-5 条活动、最近 3 个文档、最新备注、AI 建议。 |
| 趋势图 | 接入真实 `scoreHistory`，无数据时显示空态。 |

## 4. 组件规范

### 4.1 SignalCard

```
┌─────────────────────────────────────────────┐
│ ┃  标题                           [类型badge]│
│ ┃  描述                                       │
│ ┃  来自 《文档名》 · 联系人 · 2小时前          │
│ ┃  [查看文档] [忽略] [稍后] [展开分析 ▼]      │
│ ───────────────────────────────────────────  │
│ AI 解释：...                                  │
│ 建议行动：...                                 │
└─────────────────────────────────────────────┘
```

- 左侧 4px 色条标识类型。
- 类型：hot（rose）、warm（amber）、cold（slate）、risk（violet）。
- 操作：主按钮根据对象动态文案；忽略/稍后。

### 4.2 ActionList Item

```
┌─────────────────────────────────────────────┐
│ [icon] 行动标题                    [高影响]  │
│        针对 Sarah Chen · BP.pdf   [完成 ▼]   │
│        截止 6月21日 · 已逾期 1 天             │
└─────────────────────────────────────────────┘
```

- 显示关联上下文。
- 逾期高亮。
- 操作菜单：完成 / 撤销 / 推迟 / 忽略。

### 4.3 HeatMap

- 不再叫“热度地图”时使用列表形式；若用列表则改名为“活跃链接”。
- 列表只展示 hot/warm，cold 折叠。
- 每项显示：文档名、访问数、平均停留、最后访问。

### 4.4 AIAssistant

- 悬浮按钮打开面板。
- 消息区分用户/AI；AI 消息可含证据卡片。
- 快捷问题基于上下文动态生成。
- 重置需确认。

## 5. 文案规范

### 5.1 语言
- 全站中文为主，不再出现英文微文案（views、Yes/No、Week 1 等）。
- 导航、按钮、标签统一中文。

### 5.2 术语
| 英文 | 中文 |
|------|------|
| Dashboard | 交易雷达 |
| Documents | 文档库 |
| Links | 链接 |
| Deal Rooms | 数据室 |
| Contacts | 联系人 |
| Insights | 洞察 |
| Settings | 设置 |
| Hot/Warm/Cold | 高热度/中热度/低热度 |
| Risk | 风险 |

### 5.3 空态文案
- 标题简洁，描述说明原因，行动按钮明确。
- 不使用营销修辞（“360° 视图”“模板引擎”）作为页面标题。

---

## 6. 可访问性规范

- 所有交互元素可键盘访问。
- 图标按钮带 `aria-label`。
- 表单控件带正确 `label`。
- 动态内容区域带 `aria-live`。
- 支持 `prefers-reduced-motion`。

---

## 7. 验收标准

- [ ] Dashboard 首屏 5 秒内可识别高热度信号、待办行动、风险提醒。
- [ ] 任意页面首屏主要决策组 ≤ 3 个。
- [ ] 无伪造数据展示（图表、进度、分数均基于真实数据或明确空态）。
- [ ] 所有可点击元素可键盘操作。
- [ ] 全站中文文案一致，无英文混杂。
- [ ] `pnpm build` 无错误，`pnpm lint` 仅剩 TanStack Table warning。
