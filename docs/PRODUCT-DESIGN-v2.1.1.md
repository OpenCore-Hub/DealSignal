## 1. 设计哲学

### 1.1 Signal-First（信号优先）

传统文档工具以"文件"为中心，DealSignal 以"交易信号"为中心。用户打开产品首先看到：
- 今天谁表现出了高意图？
- 哪些材料正在被积极评估？
- 系统建议我下一步做什么？

### 1.2 Actionable Transparency（可行动的透明）

追踪只是起点，行动才是终点。每条数据都应有：
- **What**：发生了什么
- **So What**：这意味着什么
- **Now What**：建议做什么

### 1.3 Friction by Choice（可选的摩擦）

默认低摩擦，用户根据场景主动增加安全控制。产品必须实时显示：
- 当前安全强度
- 接收方需要完成的步骤
- 推荐配置

### 1.4 Trustworthy AI（可信赖的 AI）

AI 回答必须附带证据（evidence），证据必须可定位到原文。没有 evidence 的 AI 不可信。

### 1.5 Circle-Specific Language（圈层专属语言）

同一数据，三种 UI 表达：
- **创始人**：投资人热度、融资进度、跟进清单
- **IR**：LP engagement、合规审计、Q&A 工作流
- **销售**：Deal intent、pipeline、下一步行动

## 3. 核心模块设计

### 3.1 Dashboard 交易雷达

#### 3.1.1 布局

```
┌─────────────────────────────────────────────────────────────┐
│  Header: Workspace 名称 + 快速创建按钮                        │
├──────────────────┬──────────────────┬───────────────────────┤
│                  │                  │                       │
│  Signal Stream   │   Heat Map       │   Action Queue        │
│  (40%)           │   (35%)          │   (25%)               │
│                  │                  │                       │
│  时间倒序信号     │  文档/链接热度    │  待办跟进任务         │
│  可滑动/点击     │  颜色编码        │  一键生成邮件         │
│                  │                  │                       │
├──────────────────┴──────────────────┴───────────────────────┤
│  Risk Alerts（异常访问/过期链接/高敏下载）                    │
└─────────────────────────────────────────────────────────────┘
```

#### 3.1.2 信号流 Signal Stream

信号类型：
- **Hot Signal**：高意图行为（反复查看关键页、长时间停留、多人转发）
- **Warm Signal**：中等意图（首次打开、回访、查看多页）
- **Cold Signal**：低意图（短暂打开、跳出）
- **Risk Signal**：异常访问、过期链接、下载行为

每条信号卡片包含：
- 图标（Phosphor）
- 对象（文档/联系人/链接）
- 行为描述
- 系统解释（"这通常意味着..."）
- 建议动作（"建议 2 小时内跟进"）
- 动作按钮（写邮件 / 查看详情 / 忽略 / 稍后）

#### 3.1.3 热度地图 Heat Map

- 网格展示最近活跃的文档/链接
- 颜色：hot=red, warm=amber, cold=blue, risk=purple
- 悬停显示关键指标（views, avg duration, top page）
- 点击跳转到文档详情

#### 3.1.4 行动队列 Action Queue

- 系统基于信号生成的待办任务
- 任务卡片：优先级、建议行动、预计影响、截止时间
- 支持：完成 / 推迟 / 忽略
- 一键生成邮件草稿

### 3.2 智能链接 Smart Link

#### 3.2.1 创建器布局

```
┌─────────────────────────────────────────────────────────────┐
│  选择文档                                                    │
├─────────────────────────────────────────────────────────────┤
│  安全强度滑块                                                │
│  低摩擦 ←――――――――――――――――――――――→ 高安全                      │
├─────────────────────────────────────────────────────────────┤
│  接收方需要完成的步骤（动态更新）                             │
│  [ ] 邮箱验证  [ ] 密码  [ ] NDA  [ ] 白名单                │
├─────────────────────────────────────────────────────────────┤
│  高级选项                                                    │
│  过期时间 / 最大访问次数 / 允许下载 / 动态水印               │
├─────────────────────────────────────────────────────────────┤
│  推荐配置 + 一键复制链接 + 邮件模板                          │
└─────────────────────────────────────────────────────────────┘
```

#### 3.2.2 安全强度等级

| 等级 | 名称 | 默认配置 | 适用场景 |
|---|---|---|---|
| 1 | Open | 公开访问 | 公开 pitch deck |
| 2 | Email | 邮箱验证 | 投资人初步沟通 |
| 3 | Password | 邮箱 + 密码 | 敏感 proposal |
| 4 | Whitelist | 白名单邮箱 | 定向分发 |
| 5 | NDA | 签署 NDA | 尽调数据室 |

### 3.3 文档详情页（三位一体视图）

#### 3.3.1 布局

```
┌────────────────────────────────────┬──────────────────────┐
│                                    │                      │
│   文档预览 + 页面热力图            │   右侧标签页          │
│   (60%)                            │   (40%)              │
│                                    │                      │
│   [缩略图导航]                     │   [Overview]         │
│   [Canvas/WebP 渲染]               │   [Visitors]         │
│   [热力图叠加]                     │   [Links]            │
│                                    │   [Settings]         │
│                                    │                      │
└────────────────────────────────────┴──────────────────────┘
```

#### 3.3.2 标签页内容

- **Overview**：总览统计、热度趋势、关键页排名
- **Visitors**：访问者列表、个人时间线、热度评分
- **Links**：此文档的所有分享链接、创建新链接
- **Settings**：文档权限、版本、删除

### 3.4 联系人 360° 视图

#### 3.4.1 布局

```
┌─────────────────────────────────────────────────────────────┐
│  联系人名片（邮箱/机构/角色/热度分）                          │
├──────────────────────┬──────────────────────────────────────┤
│                      │                                      │
│  关系图              │  活动时间线                          │
│  （同机构其他访问者） │  （文档/问答/下载）                   │
│                      │                                      │
├──────────────────────┴──────────────────────────────────────┤
│  热度趋势（7/30/90 天） + 推荐行动                            │
└─────────────────────────────────────────────────────────────┘
```

### 3.5 数据室 Deal Room

#### 3.5.1 模板引擎

创建流程：
1. 选择模板（Seed/Series A/LP Update/Sales Proposal/M&A）
2. 系统自动生成文件夹结构
3. 显示推荐上传文件清单
4. 应用默认权限建议
5. 生成品牌化封面

#### 3.5.2 模板默认结构

**Seed Round Due Diligence**：
```
/ 01 Company
  - Pitch Deck
  - One Pager
/ 02 Financials
  - Financial Model
  - Cap Table
/ 03 Product
  - Product Demo
  - Roadmap
/ 04 Market
  - Market Analysis
  - Competitive Landscape
/ 05 Team
  - Founder Bios
  - Hiring Plan
/ 06 Legal
  - Incorporation
  - IP Assignment
```

### 3.6 AI 悬浮助手

#### 3.6.1 功能

- 自动摘要（打开文档时）
- 问答（附带 evidence）
- 证据链跳转（Canvas 高亮）
- 向发件人提问
- 多轮对话历史
- 角色感知高亮

#### 3.6.2 交互流程

```
用户打开文档
  ↓
AI 自动生成 3 行摘要
  ↓
用户点击悬浮按钮
  ↓
展开对话面板
  ↓
用户提问
  ↓
AI 返回 answer + evidence 列表
  ↓
默认高亮 top1 evidence
  ↓
Canvas 平滑跳转到目标页
  ↓
绘制脉冲高亮框
  ↓
用户可切换其他 evidence
```

## 4. 视觉设计规范

### 4.1 色彩系统

延续现有 Slate token：
- Primary: #0f172a
- Hot: #ef4444
- Warm: #f59e0b
- Cold: #3b82f6
- Success: #10b981
- Warning: #f59e0b
- Error: #ef4444

### 4.2 字体

- 主字体：Geist Variable
- 数字：tabular-nums

### 4.3 间距

- 页面内边距：24px（桌面）/ 16px（移动）
- 卡片间距：16px
- 组件间距：12px

### 4.4 圆角

- 卡片：12px
- 按钮：8px
- 标签：9999px

### 4.5 动效

- 默认过渡：150ms ease
- 信号流进入：stagger 50ms
- Canvas 高亮框：pulse 2s infinite
- 页面切换：fade 200ms

## 5. 响应式策略

### 5.1 桌面端（≥1024px）

- Dashboard 三栏布局
- 文档详情左右分栏
- 侧边栏常驻

### 5.2 平板端（768px-1023px）

- Dashboard 改为两栏
- 文档详情上下分栏
- 侧边栏可折叠

### 5.3 移动端（<768px）

- Dashboard 单栏卡片堆叠
- 信号流全屏可滑动
- 文档详情简化，热力图折叠
- 侧边栏抽屉

---

## 6. 无障碍要求

- 所有图表有 aria-label
- 颜色不单独传递信息（热/温/冷同时有文字标签）
- 键盘可导航
- 减少动效模式（prefers-reduced-motion）
- 对比度 ≥ 4.5:1

---

## 7. 反 AI Slop 清单

- [ ] 不使用纯装饰性大色块
- [ ] 不使用过度渐变
- [ ] 信息密度与留白平衡
- [ ] 每个 UI 元素有明确功能
- [ ] 避免" dashboard 全是图表但无行动"的反模式
- [ ] 文案使用圈层专属语言，避免通用营销话术
